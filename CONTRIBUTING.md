# Contributing to FMHY-Clients

Thank you for your interest in contributing to FMHY-Clients!

## Project Structure

```
FMHY-Clients/
├── desktop/                 # Electron desktop application
│   ├── main.js             # Electron main process
│   ├── index.html          # Desktop UI with iframe
│   └── icon.png            # Application icon (add your own)
│
├── mobile/                 # Capacitor mobile web assets
│   └── index.html          # Mobile UI with iframe
│
├── ios/                    # iOS native project (generated by Capacitor)
│   └── App/
│       ├── App.xcodeproj   # Xcode project
│       └── Podfile         # CocoaPods dependencies
│
├── android/                # Android native project (generated by Capacitor)
│   ├── app/
│   │   └── src/main/       # Android source code
│   └── build.gradle        # Gradle build configuration
│
├── capacitor.config.json   # Capacitor configuration
├── package.json            # Node.js project configuration
└── README.md               # Main documentation
```

## Development Workflow

### Making Changes to Desktop App

1. Edit `desktop/main.js` for Electron main process changes
2. Edit `desktop/index.html` for UI changes
3. Test with `npm start`
4. Build with `npm run build:win|mac|linux`

### Making Changes to Mobile Apps

1. Edit `mobile/index.html` for UI changes
2. Sync changes with `npm run cap:sync`
3. Open in IDE:
   - iOS: `npm run cap:open:ios`
   - Android: `npm run cap:open:android`
4. Build and test in respective IDE

## Code Style

- Use consistent indentation (2 or 4 spaces)
- Follow existing code patterns
- Add comments for complex logic
- Keep the Content Security Policy strict

## Security Considerations

- Always maintain proper Content Security Policy headers
- Only allow loading from trusted domains (currently https://fmhy.net)
- Disable `nodeIntegration` in Electron for security
- Keep dependencies updated

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test on relevant platforms
5. Commit with clear messages
6. Push to your fork
7. Create a Pull Request

## Testing Checklist

Before submitting a PR, ensure:
- [ ] Desktop app runs without errors
- [ ] Mobile HTML changes are synced
- [ ] No security vulnerabilities introduced
- [ ] Documentation updated if needed
- [ ] Build scripts still work

## Questions?

Feel free to open an issue for:
- Bug reports
- Feature requests
- Questions about the codebase
- Documentation improvements
